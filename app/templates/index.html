Smart Hospital
Resource Allocator

An RL agent manages ICU beds, doctors, and ambulances in real-time across an 8-hour emergency department shift. Multi-signal reward shaping. 7 discrete actions. 480-step episodes.

Environment Architecture

🧠

State Space

20-patient queue × 4 features (severity, wait-time, treatment-needed, age-risk) + resource availability + time context

⚡

Action Space

7 discrete actions: admit ICU, admit general, dispatch ambulance, escalate, discharge, call staff, hold

🎯

Reward Signal

Multi-signal shaped: survival bonus (+100), wait-time penalty, triage accuracy bonus, resource efficiency signal

🔬

OpenEnv Compliant

Full Gymnasium interface: reset(), step(), render(). Session-based API. Rate-limited. Production-grade.

Quick API Start

bash 
# 1. Start a new episode
curl -X POST http://localhost:7860/api/reset \
  -H "Content-Type: application/json" \
  -d '{}'

# 2. Take action 0 (admit to ICU)
curl -X POST http://localhost:7860/api/step \
  -H "Content-Type: application/json" \
  -d '{"session_id": "YOUR_SESSION_ID", "action": 0}'

# 3. Run a full heuristic episode (demo mode)
curl -X POST http://localhost:7860/api/demo \
  -H "Content-Type: application/json" \
  -d '{"seed": 42}'
