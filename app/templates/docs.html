API Reference

All endpoints are JSON in / JSON out. Base URL: http://localhost:7860

Overview

The API is session-based. Call /api/reset to get a session_id, then pass it in every subsequent request. Each session has its own isolated HospitalEnv instance — no shared state.

Start Episode

POST /api/reset
Start a new episode. Returns a session_id and the initial observation.
Request body (optional)
{ "seed": 42 }
Response 201
{
  "session_id": "uuid-...",
  "observation": { "patients": [...], "resources": [...], "time_context": [...] },
  "info": { "timestep": 0, "queue_length": 3, ... },
  "done": false
}

Take Action

POST /api/step
Take one action in the environment. Pass session_id in body or as X-Session-ID header.
Request body
{ "session_id": "uuid-...", "action": 0 }
Response 200
{
  "reward": 25.5,
  "done": false,
  "step_count": 1,
  "total_reward": 25.5,
  "action_name": "admit_to_icu",
  "observation": { ... },
  "info": { "patients_survived": 1, "survival_rate": 1.0, ... }
}

Heuristic Step

POST /api/heuristic-step
Let the built-in rule-based heuristic agent pick and take the next action automatically. Same response format as /step.
Request body
{ "session_id": "uuid-..." }

Run Full Episode

POST /api/demo
Run a complete 480-step heuristic episode in one shot. No persistent session. Returns full summary with hourly snapshots and per-step rewards.
Response 200
{
  "steps": 480,
  "total_reward": 1842.5,
  "survived": 38,
  "died": 2,
  "survival_rate": 95.0,
  "triage_accuracy": 88.3,
  "hourly_snapshots": [
    { "hour": 1, "queue": 4, "survived": 5, "died": 0, ... },
    ...
  ],
  "rewards": [-5.2, 28.1, ...]
}

System Status

GET /api/status?session_id=...
Server health + optional session status. session_id query param is optional.

Environment Info

GET /api/info
Static metadata: action space, observation space, episode length, reward range.

Close Session

DELETE /api/session
Explicitly free a session and its HospitalEnv resources. Pass session_id in body.

Action Space (7 discrete)

IDNameDescriptionReward Signal
0admit_to_icuAdmit queue top patient to ICU bed (needs doctor + nurse)+30 correct triage, -25 wrong
1admit_to_generalAdmit to general ward (needs nurse)+30 correct, -25 wrong triage
2dispatch_ambulanceSend ambulance; 60% chance brings critical patientResource cost penalty
3escalate_to_icuMove deteriorating general patient → ICU+30 triage correct
4discharge_patientDischarge recovered low-severity patient, frees bed+100 survival on completion
5call_extra_staff+2 doctors +3 nurses for 60 timestepsIndirect: enables more admits
6hold_and_monitorWait — monitor stable queueWait-time penalties accumulate

Observation Space

The observation is a dict with three keys:

KeyShapeRangeDescription
patientsfloat32 [80][0, 1]20 patients × 4 features: severity/5, wait/60, treatment/120, age_risk
resourcesfloat32 [5][0, 1]Normalized: ICU free, gen free, doctors, nurses, ambulances
time_contextfloat32 [2][0, 1]timestep/480, arrival rate/0.5
